# Resume Project

This directory contains my professional resume as a continuously maintained project. I treat my resume as a living document that evolves with my career growth and professional experiences.

## Overview

This project demonstrates:
- **Version Control**: Using Git to track resume updates and improvements
- **Documentation**: Maintaining clear, well-organized professional information
- **Active Maintenance**: Regular updates reflecting current skills and experiences
- **Portfolio Integration**: Resume as part of a professional portfolio repository

## Files

### `resume.md`
The primary resume file in Markdown format, making it:
- Easily viewable on GitHub
- Version-controlled and tracked for changes
- Portable and format-flexible (can be converted to PDF, HTML, etc.)
- Easy to update and maintain

## How to Use

1. **View Online**: The resume is visible directly in this GitHub repository
2. **Download**: You can download or clone the repository
3. **Convert**: Convert to PDF using tools like:
   - Pandoc: `pandoc resume.md -o resume.pdf`
   - Browser print-to-PDF
   - Online Markdown to PDF converters
4. **Share**: Share the GitHub link directly with employers or contacts

## Resume Structure

The resume includes the following sections:
- Professional Summary
- Work Experience
- Technical Skills
- Education
- Projects
- Certifications & Achievements
- Additional Information

## Updates & Changelog

All updates to the resume are tracked through Git commits. Check the commit history to see:
- When sections were added or modified
- How skills and experiences have evolved
- Updates to projects and achievements

## Why Version Control for Resumes?

✅ **Transparency**: Employers can see the evolution of your professional profile
✅ **Accountability**: Clear history of changes and improvements
✅ **Organization**: Centralized location for professional information
✅ **Flexibility**: Easy to maintain multiple versions if needed
✅ **Credibility**: Demonstrates version control and documentation skills
